# Real-Time Financial Monitor – MVP

## תיאור הפרויקט
המערכת מאפשרת קבלת טרנזקציות בזמן אמת, הצגתן בלוח Dashboard, ודמויות סימולציה להזרמת נתונים.  
המערכת כוללת Backend ב-.NET 8/9 ו-Frontend ב-React + TypeScript.

---

## בעיה בפריסה מבוזרת (Distributed Problem)
כאשר המרכיב הראשי (Backend) רץ בכמה פודים (replicas):

- כל פוד מחזיק את חיבורי ה-WebSocket שלו בלבד.
- POST שנשלח לפוד אחד (Pod A) אינו נשלח לפודים אחרים (Pod B, Pod C וכו').
- כתוצאה מכך, לקוחות שמחוברים לפודים שונים **לא יקבלו את כל העדכונים בזמן אמת**.

**דוגמה:**  
אם יש לקוח X המחובר לפוד 1 ולקוח Y המחובר לפוד 2, טרנזקציה שמתקבלת בפוד 2 תגיע ללקוח Y בלבד, והלקוח X לא ידע עליה.

---

## פתרון מוצע (Architecture Overview)
כדי לפתור את הבעיה, ניתן להשתמש ב-**Pub/Sub מרכזי** (למשל Redis):

1. **הפצת טרנזקציות (Publish):**  
   - כל פוד של ה-Backend מפרסם כל טרנזקציה שמתקבלת דרך POST לערוץ Redis משותף.
2. **קבלת טרנזקציות (Subscribe):**  
   - כל פוד מאזין לערוץ Redis ומקבל את כל העדכונים שמפורסמים מכל הפודים.
3. **שידור ללקוחות (Broadcast):**  
   - כל פוד שולח את הטרנזקציות שמתקבלות ל-WebSocket clients המחוברים אליו.

**יתרונות הפתרון:**

- כל לקוח בכל פוד יקבל את כל הטרנזקציות בזמן אמת.
- אין תלות ב-Sticky Session או חיבור ל-שרת ספציפי.
- המערכת סקיילבילית – ניתן להוסיף פודים נוספים בקלות.
- פתרון מוכר ויציב לענייני סנכרון במערכות מבוזרות.
